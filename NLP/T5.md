# T5

**Text-to-Text Transfer Transformer** — modelo encoder-decoder que reformula todas las tareas de NLP como problemas de generación de texto.

## Funcionamiento


### Origen

T5 es el antecesor de los modelos de Google y fue el primer modelo multitarea, que se podía entrenar para hacer diferentes tipos de tareas.

### Preentrenamiento

En T5, la tarea de preentrenamiento consiste en romper el texto original ocultando spans contiguos (no palabras sueltas y aisladas) y generar texto corrupto. Ese texto corrupto se le da al encoder. Luego, el decoder debe generar únicamente el contenido faltante, intercalado con tokens especiales que indican dónde fueron los huecos.

### Idea central

En vez de tener heads diferentes para cada tarea (clasificación, generación, etc.), todas las tareas se formulan como "texto a texto".

### Ejemplos

- Clasificación: "sentiment: positive" → "label: positive"
- Traducción: "translate English to German: Hallo" → "Hello"
- Resumen: "summarize: ..." → resumen generado

### Ventajas

- Un solo modelo para múltiples tareas
- Simplifica el pipeline de fine-tuning
- Escala bien con el tamaño del modelo

### Limitaciones

- No es óptimo para todas las tareas
- Requiere reformular tareas no-generativas
